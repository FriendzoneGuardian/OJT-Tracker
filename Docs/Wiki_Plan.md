# GitHub Wiki Planning: OJT-Tracker

This document outlines the proposed structure and content for the FriendzoneGuardian OJT-Tracker GitHub Wiki.

## 🎯 Wiki Goal
To provide a centralized, easy-to-navigate knowledge base for users (OJT students), IT staff (deployers), and developers (Vibecoders).

## 🗺️ Proposed Structure

### 1. 🏠 Home / Overview
- **Introduction**: Brief summary of the OJT-Tracker.
- **The "Why"**: The philosophy of accurate trajectory vs. simple division.
- **Quick Links**: Navigation to other wiki sections.

### 2. 📖 User Guide
- **Logging Hours**: How to add, edit, and delete entries.
- **Smart Trajectory**: Explanation of "Expected End" and how it adjusts.
- **Holiday & Exclusion Management**: Using the Calendar modal.
- **Customizing Your Week**: Using the new Weekend Toggles in Settings.
- **Data Export**: Guide to Excel exports.

### 3. 🛞 Deployment Guide ("On Wheels")
- **Standalone EXE**: Instructions for running without Python.
- **One-Click Launch**: Using the `run.bat`.
- **Portable Data**: How to move the `data/` folder and keep your history.

### 🛠️ Technical Reference (For Developers/Vibecoders)
- **Local Setup**: `venv` activation and `pip install` guide.
- **Architecture**: Flask/SQLAlchemy structure overview.
- **AI Agent Guidelines**: On-boarding for other coding assistants (referencing `AgentInstructions.md`).
- **Contributing**: Flow for PRs and issues.

## 📝 Next Steps
1.  **Drafting**: Create markdown files for each section in `Docs/Wiki_Drafts/`.
2.  **Implementation**: Once drafts are approved, copy-paste content into the GitHub Wiki interface.
3.  **Maintenance**: Update the Wiki with each major release.
